# 资源上传功能测试指南

## 📋 概述

本文档提供资源上传功能的完整测试指南，涵盖 H5 和微信小程序两个平台。

## 🎯 重构说明

`pages/resource/upload.vue` 已重构为使用统一的跨平台文件 API (`utils/file.ts`)，实现了：

### 重构内容

1. **文件选择** - 使用 `chooseFile()` 统一 API
   - H5: 使用 `<input type="file">`
   - 小程序: 使用 `uni.chooseMessageFile()`

2. **文件上传** - 使用 `uploadFile()` 统一 API
   - H5: XMLHttpRequest 直传 OSS
   - 小程序: uni.uploadFile 直传 OSS

3. **进度监控** - 统一的进度回调 (0-100)
   - 两端都支持实时进度更新

4. **类型安全** - 使用 `FileInfo` 接口
   ```typescript
   interface FileInfo {
     path: string      // H5: blob URL, 小程序: tempFilePath
     name: string      // 文件名
     size: number      // 文件大小（字节）
     type?: string     // MIME 类型
     raw?: File        // 原始 File 对象（仅 H5）
   }
   ```

### 文件变更

**修改的文件**:
- `frontend/src/pages/resource/upload.vue` - 资源上传页面

**修改内容**:
- ✅ 更新导入: 从 `utils/file.ts` 导入统一 API
- ✅ 重构 `chooseFile()`: 使用 `chooseFileUtil()`
- ✅ 重构 `uploadFile()`: 使用 `uploadFileUtil()`
- ✅ 更新 `handleFileSelected()`: 接受 `FileInfo` 类型
- ✅ 更新 `validateFile()`: 接受 `FileInfo` 类型
- ✅ 移除重复函数: `getFileExtension()`, `formatFileSize()`

## 🐛 已知问题和限制

### H5 平台

1. **文件选择器**:
   - 无法限制文件来源（系统、云盘等）
   - 依赖浏览器的文件选择器 UI

2. **进度监控**:
   - 依赖 `XMLHttpRequest.upload.onprogress`
   - 部分浏览器可能不支持

### 微信小程序

1. **文件选择**:
   - 只能从聊天记录中选择文件
   - 无法直接访问手机文件系统
   - 需要先发送文件到聊天（如文件传输助手）

2. **文件类型限制**:
   - 微信限制了可选择的文件类型
   - 某些格式可能无法选择

3. **上传大小限制**:
   - 微信小程序单次上传限制 10MB
   - 需要使用分片上传处理大文件 (未实现)

## 📊 性能测试

### 测试指标

1. **文件选择响应时间**:
   - H5: < 100ms
   - 小程序: < 500ms (依赖聊天记录加载)

2. **上传速度**:
   - 1MB 文件: < 2s (正常网络)
   - 10MB 文件: < 20s (正常网络)
   - 50MB 文件: < 2min (正常网络)

3. **进度更新频率**:
   - 至少 10 次/秒
   - 流畅无卡顿
