# 积分明细页面设计文档

## 一、页面概述

**页面名称**：积分明细（Points History）
**访问路径**：`/pages/user/points-history`
**功能描述**：展示用户积分变动历史记录，包括增加和减少的详细情况

## 二、UI 设计

### 1. 顶部积分概览卡片

- **当前积分**：大字号显示，突出显示
- **背景**：蓝色渐变卡片 (#2563EB → #3B82F6)
- **内容**：
  - 主标题："我的积分"
  - 积分数值：大号白色文字
  - 副标题："可用于下载资源、发布悬赏等"

### 2. 积分记录列表

每条记录显示为卡片形式：

**卡片布局**：
```
┌─────────────────────────────────────┐
│ 📍 变化原因           +10 / -5     │
│ 变化后积分: 150                     │
│ 2024-01-20 14:30                   │
└─────────────────────────────────────┐
```

**字段说明**：
- **左上角图标**：根据变化类型显示不同图标
  - 增加 (正数)：绿色 `+` 号
  - 减少 (负数)：红色 `-` 号
- **变化原因**：如"上传资源"、"下载资源"、"回答被采纳"等
- **积分变化**：
  - 正数显示为绿色 `+N`
  - 负数显示为红色 `-N`
- **变化后积分**：灰色小字
- **时间**：格式化显示（今天/昨天/具体日期）

### 3. 积分类型图标和颜色规范

| 原因类型 | 图标 | 颜色 | 示例 |
|---------|------|------|------|
| 注册奖励 | 🎁 | 绿色 | +100 |
| 上传资源 | 📤 | 绿色 | +10 |
| 下载资源 | 📥 | 红色 | -5 |
| 提问 | ❓ | 红色 | -2 |
| 回答问题 | 💬 | 绿色 | +5 |
| 回答被采纳 | ✅ | 绿色 | +20 |
| 发布任务 | - | 红色 | -悬赏 |
| 完成任务 | 🎯 | 绿色 | +悬赏 |
| 活动签到 | ✔️ | 绿色 | +10 |
| 每日签到 | 📅 | 绿色 | +10 |

### 4. 空状态

- **图标**：💰 (大号)
- **文案**："暂无积分记录"
- **提示**："完成任务、上传资源等可以获得积分哦~"

### 5. 加载状态

- 首次加载：显示加载文本
- 加载更多：底部显示"加载中..."
- 无更多数据：显示"没有更多了"

## 三、交互设计

### 1. 下拉刷新

- 支持下拉刷新最新积分记录
- 刷新时同时更新顶部当前积分数值

### 2. 上拉加载更多

- 滚动到底部自动加载下一页
- 每页显示 20 条记录
- 使用分页参数：`page`、`pageSize`

### 3. 卡片点击

- 如果有关联对象（relatedType 和 relatedId）：跳转到对应详情页
  - `resource` → 资源详情页
  - `question` → 问答详情页
  - `task` → 任务详情页
- 如果没有关联对象：不响应点击

## 四、数据接口

### API 端点

```
GET /user/points/log
```

### 请求参数

| 参数 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| page | Integer | 否 | 1 | 页码 |
| pageSize | Integer | 否 | 20 | 每页数量 |

### 响应数据

```typescript
interface PointsLog {
  logId: number
  userId: number
  pointsChange: number        // 正数为增加，负数为减少
  pointsAfter: number         // 变化后的积分
  reason: string              // 变化原因
  relatedType?: string        // 关联对象类型
  relatedId?: number          // 关联对象ID
  createdAt: string          // 创建时间
}

interface PageResult<T> {
  list: T[]
  total: number
  page: number
  pageSize: number
  totalPages: number
}
```

## 五、技术实现要点

### 1. 时间格式化

复用收藏页的 `formatTime()` 函数：
- 今天：显示 "今天 HH:mm"
- 昨天：显示 "昨天 HH:mm"
- 本年：显示 "MM-DD HH:mm"
- 往年：显示 "YYYY-MM-DD"

### 2. 积分变化颜色

```typescript
const getPointsColor = (change: number) => {
  return change > 0 ? '#10B981' : '#EF4444'  // 绿色/红色
}

const getPointsText = (change: number) => {
  return change > 0 ? `+${change}` : `${change}`
}
```

### 3. 原因类型映射

```typescript
const getReasonIcon = (reason: string): string => {
  const iconMap: Record<string, string> = {
    '注册奖励': '🎁',
    '上传资源': '📤',
    '下载资源': '📥',
    '提问': '❓',
    '回答问题': '💬',
    '回答被采纳': '✅',
    '完成任务': '🎯',
    '活动签到': '✔️',
    '每日签到': '📅'
  }
  return iconMap[reason] || '💰'
}
```

### 4. 分页逻辑

- 初始加载第 1 页
- 下拉刷新：重置为第 1 页
- 上拉加载：page++
- 根据 totalPages 判断是否还有更多

2. 积分变化动画效果
