# CampusLink Design System 1.0

官方设计规范文档
（CampusLink 全端统一设计系统）

---

# 1. 品牌理念（Brand Core）

### 品牌关键词

* 校园
* 青年
* 可信赖
* 学习成长
* 清爽科技
* 社交互动

### 品牌愿景

打造高校中最清爽、最专业、最值得信任的互助与学习互动平台。

### 设计哲学

**轻品牌、强秩序、弱装饰、强引导。**
强调信息优先级与阅读体验，不使用过度 AI 风，不采用花哨插画。

---

# 2. Design Tokens（核心基础变量）

这些 Token 是全站设计语言的基石，所有组件、页面、海报均基于此。

---

## 2.1 色彩系统（Color System）

### ① 主品牌色（Campus Blue）

| 颜色          | 十六进制        | 用途        |
| ----------- | ----------- | --------- |
| Primary 500 | **#377DFF** | 按钮、链接、主操作 |
| Primary 400 | #5A8BFF     | Hover 态   |
| Primary 300 | #8FB2FF     | 辅助色、背景点缀  |
| Primary 50  | #EFF5FF     | 浅背景区、卡片底衬 |

### ② 成功 / 状态色

| 类型      | Hex         | 用途       |
| ------- | ----------- | -------- |
| Success | **#3CCB7F** | 已解决、成功状态 |
| Warning | **#FFB766** | 活动已结束、提醒 |
| Danger  | **#FF5C5C** | 删除、错误    |

### ③ 中性色（企业级视觉体系）

| 名称         | Hex     | 用途           |
| ---------- | ------- | ------------ |
| Title      | #1A1A1A | 主标题          |
| Body       | #3D3D3D | 正文           |
| Secondary  | #6B778C | 次要说明、Meta 信息 |
| Border     | #E6E6E6 | 分割线          |
| Background | #FAFBFC | 页面背景         |

---

## 2.2 字体体系（Typography）

### PC / App / 小程序统一采用

**HarmonyOS Sans（默认） / PingFang SC / Inter（英文字体）**

### 字体层级（Font Scale）

| 层级   | 字号(px) | 字重  | 用途                |
| ---- | ------ | --- | ----------------- |
| H1   | 24     | 600 | 模块标题（精选推荐 / 精选资料） |
| H2   | 20     | 600 | 卡片标题、活动标题         |
| H3   | 18     | 500 | 问答标题、小标题          |
| Body | 14–16  | 400 | 正文                |
| Meta | 12     | 400 | 时间、浏览数、积分等元数据     |

说明：

* 全站标题字重统一为 **600**
* Meta 信息统一 **12px + #6B778C**
* 不使用 500/700 以外的奇数字重，保持统一性

---

## 2.3 圆角体系（Border Radius）

| 类型     | 圆角值      | 用途         |
| ------ | -------- | ---------- |
| Card   | **16px** | 所有主卡片      |
| Button | 8–12px   | 小程序/APP圆润感 |
| Tag    | 6px      | 标签统一       |
| 图标容器   | 12px     | 头像 / 图标背景  |

---

## 2.4 阴影体系（Shadows）

符合企业级平台的轻稳风格。

| 名称           | 阴影值                            | 用途       |
| ------------ | ------------------------------ | -------- |
| Card Shadow  | `0 4px 20px rgba(0,0,0,0.04)`  | 正常卡片     |
| Hover Shadow | `0 6px 24px rgba(0,0,0,0.08)`  | PC Hover |
| Modal Shadow | `0 12px 40px rgba(0,0,0,0.12)` | 弹窗       |

---

# 3. 栅格系统（Grid System）

### 基础原则

* 左侧主列宽度保持一致
* 右侧侧栏宽度固定
* 所有卡片左边距 24px 对齐
* 卡片模块间距统一为 **24px**

### PC 端（≥1280px）

| 区域     | 宽度         |
| ------ | ---------- |
| 主内容区   | 720–780 px |
| 辅助栏    | 320–360 px |
| 全局左右边距 | 48–64 px   |

### H5 / 小程序

采用固定左右 16px 安全边距：

```
| 16px | 内容区域 | 16px |
```

# 4. 组件系统（CampusLink Components）

这是未来开发和设计通用的组件库规范。

---

## 4.1 按钮（Button）

### 主按钮 Primary

* 背景：`#377DFF`
* 字体：14px / 600 / #FFF
* 圆角：12px
* 阴影（可选）：`0 2px 6px rgba(55,125,255,0.22)`

### 次按钮 Secondary

* 边框：1px solid #377DFF
* 字体颜色：主蓝
* 背景：透明

### 安全按钮 Safe

* 背景：#3CCB7F

### 危险按钮 Danger

* 背景：#FF5C5C

---

## 4.2 标签（Tags）

标签统一 6px 圆角 + 12px 字号。

| 类型     | 背景        | 字体            |
| ------ | --------- | ------------- |
| 默认 TAG | #EFF5FF   | #377DFF       |
| 热门 HOT | #FFECEC   | #FF5C5C       |
| 状态标签   | 深色背景 + 白字 | 如 “已解决”、“报名中” |

---

## 4.3 卡片系统（核心统一化成果）

四大卡片保持相同结构：

```
<Card>
  Header
  Body
  Meta
  Action
</Card>
```

所有卡片采用：

* 圆角：16px
* 阴影：Card Shadow
* 左对齐：24px
* 模块间距：16px

---

## 4.4 Meta 信息（浏览 / 评论 / 时间）

统一样式规范：

* 图标大小：16–18px
* 颜色：`#6B778C`
* 行高：16px
* 排列方式：水平 8px 间距

---

# 5. 图标系统（Icons）

### 核心原则

* 全站统一采用 **线性图标（Linear 2px）**
* 主要参考：Lucide、Remix Icon、IconPark 线性系
* 图标统一大小：18px（小）/ 24px（中）

图标圆角统一遵循：

* 容器：12px
* 图标 stroke：2px

不使用填充图标，避免风格杂乱。

---

# 6. 插画风格（Illustration System）

CampusLink 插画需符合：

### ① 轻科技校园风（不 AI 风、不儿童风）

# 7. 多端适配规范（Web / H5 / 小程序 / App / 鸿蒙）

### Web PC

* 栅格固定 1280–1440
* Hover 态必须有阴影或背景变化

### H5

* 按钮高度 ≥44px
* 点击区域扩大至最少 44×44px
* 去除 hover，保留 active 态

### 小程序（微信）

* 定义 `activeOpacity: 0.7`
* Tag 圆角保持 6px
* 资源卡片、问答卡片统一默认白底无边框

### App（uni-app）

* 提升投影（APP 端更能表现阴影）
* 图标自适应 `dpr`

### 鸿蒙元服务

* 使用 HarmonyOS Sans
* 保留卡片与控件统一圆角体系（更适用于元服务）
