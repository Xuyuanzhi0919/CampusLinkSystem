## 📈 积分系统设计冲突分析

### 原设计文档的问题

**原 `resource-upload-page-design.md` (第 151 行)**:
```markdown
| score | 数字输入框 | ✅ | 1-20 | 3 | 下载所需积分 |
```

**矛盾点**:
1. 允许用户自定义下载积分（1-20）
2. 但系统统一规定"下载资源 -5 分"
3. 两者冲突！

### MVP 设计的正确修正

**`resource_upload_mvp1.md` 明确**:
> 本次明确不做：积分自定义/资源定价相关设置

**统一积分规则**:
- 上传资源（审核通过）: +10 分
- 下载资源: -5 分（**固定**，不允许自定义）

### 当前实现状态: ✅ 符合 MVP

- ❌ 前端未暴露 `score` 字段
- ✅ 后端自动设置 `score = 5`
- ✅ 与积分系统规则一致
