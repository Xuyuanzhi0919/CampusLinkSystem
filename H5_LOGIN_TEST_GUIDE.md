### 步骤 3：验证登录流程

#### 3.1 观察浏览器 Console 日志

按 `F12` 打开开发者工具，在 Console 面板中应该看到：

```
[H5登录] 开始登录... { account: 'admin', password: '******' }
[H5登录] 登录成功，用户信息: { uid: 1, username: 'admin', ... }
```

#### 3.2 观察 Network 面板

在 Network 面板中查找 `/api/v1/auth/login` 请求：

**请求（Request）：**
```json
POST /api/v1/auth/login
Content-Type: application/json

{
  "account": "admin",
  "password": "admin123"
}
```

**响应（Response）：**
```json
{
  "code": 200,
  "message": "success",
  "data": {
    "token": "eyJhbGc...",
    "refreshToken": "eyJhbGc...",
    "user": {
      "uId": 1,
      "username": "admin",
      "nickname": "管理员",
      "role": "admin",
      "points": 100,
      "level": 1,
      ...
    }
  },
  "timestamp": 1766025000000
}
```

#### 3.3 验证 Token 存储

在 Console 中执行：
```javascript
console.log('Token:', localStorage.getItem('campuslink_token'))
console.log('RefreshToken:', localStorage.getItem('campuslink_refresh_token'))
console.log('User:', localStorage.getItem('campuslink_user'))
```

应该能看到 Token 和用户信息已保存。

#### 3.4 确认页面跳转

登录成功后，应该：
1. 显示 "登录成功" 提示（绿色 Toast）
2. 1.5 秒后自动跳转到首页（`/pages/home/index`）
3. 用户中心显示已登录状态

## 🧪 测试场景

### 场景 1：正常登录 ✅

1. 输入正确的用户名和密码

**预期结果：**

2. 输入任意密码

**预期结果：**
- ❌ 显示"用户名或密码错误"或"账号不存在"提示
- ❌ 保持在登录页面

---

### 场景 6：快速重复点击（防抖测试）⏱️

**操作：**
1. 输入正确的账号密码
2. 快速连续点击"登录"按钮 5 次

**预期结果：**
- ✅ 第一次点击后按钮变为"登录中..."状态
- ✅ 后续点击无效（按钮禁用）
- ✅ 只发送一次 API 请求

### 场景 7：登录状态持久化 💾

**操作：**
1. 成功登录
2. 刷新页面（F5）

**预期结果：**
- ✅ 刷新后仍然保持登录状态

- ✅ 不需要重新登录

### 场景 8：退出登录后重新登录 🔄

**操作：**
1. 登录成功
2. 进入用户中心
3. 点击"退出登录"
4. 重新访问登录页面
5. 再次登录

**预期结果：**
- ✅ 退出后 localStorage 已清空
- ✅ 可以重新登录
- ✅ 新 Token 正常保存

## 🐛 常见问题排查

### 问题 1：页面无法访问（404）

**可能原因：**
- 登录页面未添加到 `pages.json`
- H5 开发服务器未启动

**解决方案：**
```bash
# 检查 H5 服务器是否运行
netstat -ano | findstr :5173

# 如果没有运行，启动它
cd frontend
npm run dev:h5
```

---

### 问题 2：登录请求 404

**可能原因：**
- 后端服务未启动
- API 路径错误

**解决方案：**
```bash
# 检查后端服务是否运行
netstat -ano | findstr :8080

# 如果没有运行，启动它
cd backend
mvn spring-boot:run

# 测试 API 是否可访问
curl http://localhost:8080/api/v1/health
```

---

### 问题 3：登录请求返回 401

**可能原因：**
- 登录 API 被认证拦截器拦截了

**解决方案：**
检查 `WebConfig.java` 中是否添加了 `/auth/login` 到白名单：
```java
.excludePathPatterns(
    "/auth/register",
    "/auth/login",        // ← 必须存在
    "/auth/refresh",
    "/auth/wechat/login"
)
```

---

### 问题 4：用户名或密码错误

**可能原因：**
- 测试账号不存在
- 密码不正确

**解决方案：**
在数据库中查询用户：
```sql
-- 查询管理员账号
SELECT u_id, username, role, points FROM user WHERE username = 'admin';

-- 如果不存在，创建测试账号
-- （需要先通过注册接口或直接插入数据库）
```

---

### 问题 5：Token 无法保存

**可能原因：**
- localStorage 被浏览器禁用
- 隐私模式/无痕模式

**解决方案：**
1. 退出隐私模式/无痕模式
2. 检查浏览器设置，允许网站存储数据
3. 在 Console 中手动测试：
   ```javascript
   localStorage.setItem('test', '123')
   localStorage.getItem('test')  // 应该返回 '123'
   ```
