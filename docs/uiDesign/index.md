0）全局基调（先定风格再细抠）

目标气质： 学术清爽（Campus）× 轻科技（AI）
关键词： 统一、克制、呼吸、顺畅

色彩：

主色 #2563EB（比现有蓝更稳）、辅助紫 #6C5CE7 只用于强调；

点缀橙 #F59E0B 用于“答/提醒”，饱和度收 10%；

页面背景 #FAFAFC，卡片 #FFFFFF，分隔线 #EAEAEA。

字体： 中文 MiSans / 思源黑体；英文 Inter。

H1 22–24 / 600，H2 18 / 600，正文 14–16 / 400，说明 12 / 400。

圆角与阴影： 圆角体系 16（容器）/12（卡片）/10（控件）；

Elev.1：0 2px 8px rgba(0,0,0,.05)

Elev.2：0 6px 20px rgba(0,0,0,.08)

栅格与间距： 容器内边距 24；模块间距 32–40；卡片间距 16；行间距 12–16。

动效： 入场 240ms ease-out；Hover 150ms；切换 200ms，统一曲线 cubic-bezier(.2,.8,.2,1)。

可访问： 文字与底色对比≥4.5:1；可点击区≥44×44；焦点环 2px 主色 35% 透明。

1）顶部 Hero/导航区

当前问题

蓝底插画占比过大，视觉重心在顶部而不是内容区；

搜索条太“白亮”，与背景对比过强；

校园徽章、提示问号、右侧人物插画视觉噪点偏多。

优化

缩短 Hero 高度到 200–240px；背景换为浅蓝渐变（非饱和亮蓝），插画弱化透明到 12–16%。

搜索条半透明玻璃：rgba(255,255,255,.75) + blur(8px)，圆角 16，描边 rgba(0,0,0,.06)；按钮蓝 #2563EB，语音橙降饱和。

信息密度：保留 Logo、口号、搜索；其它徽章/人物插画移到 Banner 右上角淡淡出现或直接删除。

下边界加 1px 分隔线，给内容区“落地”。

2）功能宫格（资料共享 / 智能问答 / 互助任务）

当前问题

三块卡视觉重量不一致（中间橙块太抢），插图过实；

卡片说明文案长度不一，导致纵向节奏乱。

优化

改为等宽卡，高度一致；背景浅色+细腻渐变（非纯块色）。

插画改“线性/双色矢量”，透明度 18–22%；

标题 18/600；副文 12/400，限制单行省略；

Hover：微浮起 + 阴影增强，标题主色高亮。

三卡使用同一内边距（24），对齐图标与文字基线。

3）快捷入口条（社团活动/附近同学/积分商城/公告/收藏）

当前问题

每个胶囊块大小不一致；图标风格混杂（emoji/3D/线性）。

优化

统一为胶囊按钮：高度 44、左右内边距 12、图标 18、图标与文案间距 8；

图标统一线性或双色；默认灰 600，Hover 主色。

横向排列时保持等间距 12；整条放在浅底条上（#F5F7FB），让区域成组出现。

4）主内容：为你推荐（2×3 网格）

当前问题

卡片样式“轻”和“平”，标题与描述权重接近；

标签浅+边距不统一；底部互动组件（点赞/收藏）视觉噪点偏多；

列左右卡片高度差大，流失秩序感。

优化

数量：桌面 6 条（2×3），卡片高度最小 168，描述行数最多 2 行。

卡片层级

顶部类型条（课件/问答/任务）：浅底 + 8px 圆角，色板统一

课件 #E0F2FE / #3B82F6

问答 #FEF3C7 / #F59E0B

任务 #DCFCE7 / #16A34A

标题 16–18/600，颜色 #0F172A；

描述 14/400，颜色 #64748B；

元信息（作者/浏览/下载）统一灰 500；图标线性 1.5px。

交互反馈

Hover：translateY(-3px) + 蓝柔影 0 8px 24px rgba(37,99,235,.08)；

点击：边框变主色 1px；

互动按钮组合并成一个“收藏/点赞”分组，减少噪点。

“换一批/查看全部”条

放在网格下方独立卡槽，浅蓝底 #EEF2FF，圆角 12；

点击“换一批”时交叉淡入（卡片 stagger 30ms）。

5）右侧栏：热门问答（含 Tab）

当前问题

侧栏视觉存在感弱、留白大；排名与“答”按钮尺寸偏小；

Tab 下划线较细，激活感不足。

优化

容器：右栏加浅阴影与更深的表面色 #FBFBFE，与左侧拉开层级；内边距 20–24。

条目：每项高度 56，左侧排名胶囊（1 蓝、2 橙、3 绿，深色文字）；右侧固定“答”按钮 28 高度。

Tab：粗 3px 的主色滑块，切换动效 200ms；

补充行：在标题下加入“8 回答 · 203 浏览”这种副信息。

查看完整榜单按钮：描边主色、Hover 渐入主色填充。

6）FAB 悬浮菜单（右下角蓝圆）

当前问题

与页面其它蓝色元素竞争注意力；位置略靠内，遮挡列表时会心烦。

优化

FAB 直径 56，离右/下边 16；

遮挡列表时自动下移或缩小（滚动隐藏、上滑出现）；

展开采用 Speed Dial：向上弹出 3–4 个常用动作；

动效：200ms scale-in + 20ms 阶梯延迟；背景蒙层 rgba(14,19,32,.18)。

7）三张“登录空状态卡片”（积分中心/公告/社团）

当前问题

模块彼此割裂、按钮风格不统一；插画与文字权重不均。

优化

置于同一浅色容器区（如 #F7F8FC），三卡同宽同高（最小 240），圆角 16；

插画风格统一（线性/双色），透明 18%；

主文案 16/600（“登录后解锁…”），副文案 12/400；

按钮统一主色蓝（或蓝紫渐变），Hover 发光。

加顶部小角标（🔒）提示“需登录”，减少反复阅读成本。

8）视觉噪点清除清单

统一图标风格：全部改线性（IconPark/Lucide），不要混用 emoji/3D。

收敛渐变与发光：只在主按钮和Hero 背景上保留；

删除“装饰性小涂鸦”或把不相关插画透明度降到 ≤10%；

文案统一句式与标点（不要 “·” 和 “/” 混用）；

所有数值项（下载/浏览/回答）统一顺序与分隔符。

9）响应式与密度策略

≥1280：左 68% / 右 30% / 中间间距 2%；

1024–1279：右栏收窄，条目改为 4 条；

<960：右栏下沉为分页模块；网格改单列 + 4 条展示；

移动端“查看全部”变为满宽胶囊按钮，FAB 保留。
