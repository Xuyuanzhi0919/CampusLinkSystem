# 资源下载功能测试指南

## 📋 概述

本文档提供资源下载功能的完整测试指南,涵盖 H5 和微信小程序两个平台。

## 🎯 重构说明

`pages/resource/detail.vue` 的下载功能已重构为使用统一的跨平台文件 API (`utils/file.ts`)。

### 重构内容

1. **文件下载** - 使用 `downloadFile()` 统一 API
   - H5: 创建 `<a>` 元素触发浏览器下载
   - 小程序: 使用 `uni.downloadFile()` + `uni.openDocument()`

2. **自动平台适配**:
   - H5: 新窗口打开或直接下载
   - 小程序: 下载到临时目录后使用系统应用打开

3. **文件名处理**:
   - 使用资源标题作为文件名
   - 自动从 URL 获取文件扩展名

### 文件变更

**修改的文件**:
- `frontend/src/pages/resource/detail.vue` - 资源详情页

**修改内容**:
- ✅ 添加导入: `import { downloadFile as downloadFileUtil } from '@/utils/file'`
- ✅ 重构 `confirmDownload()` 函数:
  - 移除平台条件编译代码
  - 移除 `uni.downloadFile()` 直接调用
  - 使用 `downloadFileUtil()` 统一 API
  - 代码从 48 行简化到 35 行

## 🐛 已知问题和限制

### H5 平台

1. **浏览器兼容性**:
   - Safari 可能拦截新窗口弹出（安全策略）
   - 需要用户允许弹出窗口

2. **文件名问题**:
   - 某些浏览器可能不支持自定义文件名
   - 文件名可能被 URL encode

3. **大文件下载**:
   - 超大文件（>100MB）可能导致浏览器卡顿
   - 建议使用分片下载（未实现）

### 微信小程序

1. **文件类型限制**:
   - 部分文件类型无法预览（如 .rar, .7z）
   - 只能下载到微信临时目录

2. **临时文件管理**:
   - 微信会定期清理临时文件
   - 用户无法手动管理下载文件

3. **文件大小限制**:
   - 微信限制单个文件下载最大 50MB
   - 超大文件需要分片下载

## 📊 性能测试

### 测试指标

1. **下载速度**:
   - 1MB 文件: < 2s (正常网络)
   - 10MB 文件: < 10s (正常网络)
   - 50MB 文件: < 1min (正常网络)

2. **积分扣除响应**:
   - 后端扣除积分: < 500ms
   - 前端状态更新: 实时

3. **错误恢复**:
   - 网络错误后重试: 自动（依赖浏览器/微信）
   - 积分回滚: 不支持（后端已扣除则不回滚）
