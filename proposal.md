# 本科毕业设计（论文）开题报告

## 一、选题背景与意义

### 1.1 选题背景

当前，高等教育数字化转型已成为教育改革的重要方向。根据教育部发布的《教育信息化2.0行动计划》，构建"互联网+教育"新生态、推动优质教育资源共享是新时代高等教育发展的核心议题。然而，在实际的高校场景中，学习资源的共享与知识交流面临诸多困境：

**（1）资源分散问题**：学习资料分散于各类即时通讯群组（如微信群、QQ群），缺乏统一的沉淀与检索机制，优质资料随聊天记录沉没，难以被后续学习者发现和复用。

**（2）问答效率问题**：课程疑难问题的解答效率偏低，同一问题在不同学期被反复提问，优质答案难以被系统化沉淀，缺乏有效的知识问答社区平台。

**（3）校园协作问题**：校园内的勤工助学、技能互助等任务需求缺乏结构化的撮合平台，信息不对称导致供需双方难以高效匹配。

**（4）社团活动问题**：社团活动与校园文化活动的组织与传播渠道单一，参与度难以提升，活动信息触达率低。

基于上述背景，本课题拟设计并实现一套名为CampusLink的高校资源共享与问答社区平台，以H5技术为载体，充分利用手机浏览器的普及性，构建一个集资源共享、知识问答、任务协作、社团活动管理于一体的高校垂直社区。

### 1.2 研究意义

**理论意义**：本课题探索了前后端分离架构在高校垂直场景下的工程实践，验证了Spring Boot与uni-app技术栈在校园社区系统中的适用性，对同类系统的设计与实现具有一定参考价值。

**实践意义**：
- 为高校学生提供专属的数字化学习协作工具，提升学习资源共享效率
- 通过积分激励机制，形成正向循环的社区生态，鼓励知识贡献
- 引入AI辅助答题，提升知识获取效率
- 为校园内的技能互助行为提供结构化的协作规范

## 二、国内外研究现状

### 2.1 国外研究现状

**技术问答平台方面**，以Stack Overflow为代表的技术问答平台在问题结构化、回答质量评估机制上积累了丰富的经验，其采纳答案、投票、徽章等激励手段已被广泛借鉴。

**在线教育平台方面**，Coursera、edX等MOOC平台在课程资源的组织与分发层面提供了优秀的范本，支持视频课程、作业提交、讨论区等功能。

**社区平台方面**，Reddit、Discord等社区平台探索了不同粒度的知识共同体运营模式，支持子版块、实时聊天等功能。

然而，上述平台均定位于通用场景，缺乏对高校特定业务流程（如课程归属、学号验证、社团管理、校园任务）的原生支持。

### 2.2 国内研究现状

**教育类应用方面**，国内已有若干高校知识服务类应用，如超星学习通、钉钉校园版等，侧重于课程管理与师生互动，但其问答与资源共享功能较弱，且封闭性强，难以形成跨课程、跨年级的知识社区氛围。

**校园论坛方面**，部分高校自研了校内BBS或论坛系统，但技术栈较为陈旧（多采用PHP、JSP等传统技术），移动端体验欠佳，缺乏现代化的用户体验设计。

**技术研究方面**，已有研究者探索了基于Spring Boot的高校教学资源平台建设，以及采用Vue.js与Spring Boot前后端分离架构的校园应用开发。赵明文等人设计的高校学习社区系统在功能定位上与本项目存在一定交叉，但缺乏任务大厅、积分激励及AI辅助问答等核心特性。

### 2.3 研究现状总结

综上所述，目前缺乏一套能够同时满足以下需求的轻量级综合平台：
- 资源共享与知识问答
- 校园任务协作
- 社团活动管理
- 积分激励机制
- AI智能辅助
- 良好的移动端体验

这正是本系统的设计动机与价值定位。

## 三、研究内容与目标

### 3.1 研究内容

本课题主要研究内容包括：

**（1）需求分析**：深入调研高校师生的实际需求，完成功能性需求与非功能性需求的系统梳理，形成完整的需求规格说明书。

**（2）系统架构设计**：设计前后端分离的系统架构，完成技术选型、数据库表结构设计及API接口规范设计。

**（3）核心功能模块实现**：
- 用户认证与授权模块（JWT双令牌机制）
- 资源共享模块（OSS直传、审核流程）
- 问答社区模块（Markdown编辑、AI辅助答题）
- 任务大厅模块（状态机设计、信用评分）
- 积分激励模块（积分规则、等级系统）
- 社团活动模块（活动发布、报名签到）
- 管理后台模块（用户管理、内容审核、数据统计）

**（4）关键技术攻关**：
- JWT双令牌并发刷新防竞争机制
- WebSocket实时消息推送
- 阿里云OSS文件直传
- DeepSeek AI接口集成

**（5）系统测试与评估**：完成功能测试、性能测试，验证系统在预期负载下的响应能力与稳定性。

### 3.2 研究目标

**功能目标**：实现一个功能完整的高校资源共享与问答社区平台，涵盖资源共享、问答社区、任务大厅、社团活动、积分激励及管理后台六大核心模块。

**性能目标**：核心接口（列表查询、详情获取）的响应时间在正常负载下不超过200ms；系统支持至少100名用户并发访问而不发生服务降级。

**用户体验目标**：H5用户端具备良好的移动端适配，首屏加载时间控制在2秒以内；管理后台操作简洁高效，数据可视化直观清晰。

## 四、研究方法与技术路线

### 4.1 研究方法

**（1）文献研究法**：查阅国内外关于高校资源共享平台、问答社区系统、前后端分离架构等方面的文献资料，了解相关领域的研究现状与技术发展趋势。

**（2）需求调研法**：通过问卷调查、用户访谈等方式，深入了解高校师生对学习资源共享与知识交流的实际需求，为系统设计提供依据。

**（3）原型设计法**：使用Figma等工具设计系统原型，通过用户评审验证设计方案的合理性。

**（4）敏捷开发法**：采用迭代式开发模式，将系统功能拆分为多个迭代周期，每个周期完成一部分功能的开发与测试。

### 4.2 技术路线

#### 4.2.1 后端技术栈

|---------|------|------|
| Spring Boot | 3.4.0 | 后端框架 |

| MyBatisPlus | 3.5.9 | ORM框架 |
| MySQL | 8.0+ | 关系型数据库 |
| Redis | 7.0+ | 缓存与会话存储 |

| WebSocket | - | 实时通信 |
| Knife4j | - | API文档 |
| 阿里云OSS | - | 对象存储 |
| DeepSeek API | - | AI辅助 |

#### 4.2.2 前端技术栈

| 技术组件 | 版本 | 用途 |
|---------|------|------|
| uni-app | - | 跨平台框架 |
| Vue 3 | - | 前端框架 |
| TypeScript | - | 类型支持 |

| Pinia | - | 状态管理 |
| Element Plus | - | 管理后台UI |

#### 4.2.3 系统架构

系统采用经典的前后端分离架构，整体划分为客户端层、接入层、服务层和数据层四个层次，如图4-1所示。

![图4-1 CampusLink系统总体架构图](thesis-figures/09_architecture.png)

**图4-1** CampusLink系统总体架构图

**架构说明**：

- **客户端层**：包含H5用户端（uni-app，端口5173）与Vue 3管理后台（端口5174）
- **接入层**：Nginx反向代理，实现负载均衡与HTTPS终止
- **服务层**：Spring Boot应用，统一路径前缀 `/api/v1`，包含REST API与WebSocket端点
- **数据层**：MySQL 8.0主数据库、Redis 7.0缓存、阿里云OSS文件存储

## 七、可行性分析

### 7.1 技术可行性

**（1）成熟的技术栈**：Spring Boot、Vue 3、MySQL等技术均为业界成熟方案，有丰富的文档和社区支持。

**（2）个人技术储备**：本人已系统学习Java Web开发、前端开发等相关课程，具备Spring Boot、Vue.js等技术的开发经验。

**（3）硬件环境**：开发所需硬件（个人电脑、云服务器）均已具备。

### 7.2 经济可行性

**（1）开发成本**：所有开发工具和框架均为开源免费，无需购买商业软件许可。

**（2）运行成本**：可使用阿里云学生优惠服务器，成本可控。

**（3）人力成本**：由本人独立完成开发，无额外人力成本。

### 7.3 操作可行性

**（1）用户习惯**：平台采用H5技术，无需安装APP，符合高校用户的使用习惯。

**（2）学习成本**：界面设计简洁直观，用户无需培训即可上手使用。

**（3）维护成本**：采用前后端分离架构，便于后期维护和功能扩展。
